Reports Java and Kotlin classes annotated with @ComponentView that have embedded FXML markup but no sibling .fxml file, and offers to extract the markup into a standalone FXML file, removing the annotation.

Use Fix all in file or Fix all in scope to create FXML files for multiple annotated classes at once.

This inspection is disabled by default. When disabled, the Create FXML file intention action is still available at individual caret positions.

Reports invalid or duplicate {@code } processing instructions in FXML 2.0 files.

The fxml2 compiler allows single-character prefix shorthands to be declared before the root element, mapping a prefix character to a markup-extension class: 

  <?prefix ^ = org.example.MyExtension?>
Reported problems:
  • Invalid prefix character: letters, digits, {@code _}, {@code $}, whitespace, and reserved punctuation characters are not allowed.
  • Duplicate declaration for the same prefix character.
  • Unresolvable class name in the declaration.

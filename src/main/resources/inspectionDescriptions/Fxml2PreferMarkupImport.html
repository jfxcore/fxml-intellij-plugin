Reports Java or Kotlin import statements that are only used inside embedded FXML markup (@ComponentView annotation values) and never in the surrounding code.

When all usages of an imported class are confined to the embedded markup string, the import can be expressed as a <?import fqn?> processing instruction directly inside the markup block, keeping the host file's import section free of markup-only dependencies.

The quick-fix Move to markup <?import?> inserts the corresponding <?import?> PI into every @ComponentView markup block in the file and removes the Java or Kotlin import statement.

This inspection is disabled by default. Enable it when your project prefers all class references used exclusively in embedded markup to be declared as <?import?> PIs rather than as top-level code imports.

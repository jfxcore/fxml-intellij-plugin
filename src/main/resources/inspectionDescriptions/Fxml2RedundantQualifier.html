Reports fully-qualified class names whose package prefix is redundant because the class is already accessible via an <?import?> declaration.

Affected locations:

  • fx:typeArguments attribute values
  • FQN element tag names (e.g. <javafx.scene.control.TextField/>)
  • Binding-expression paths with a FQN static-field class prefix (e.g. $javafx.scene.layout.Region.USE_PREF_SIZE)

A quick-fix removes the redundant prefix, leaving just the simple (or shortest resolvable) name.

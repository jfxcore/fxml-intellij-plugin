Reports a redundant !! operator in an FXML binding expression when the resolved type of the bound path is the boolean type.

The FXML compiler treats !! on a boolean-typed observable as a compile-time no-op: the observable is passed directly to the target property; the !! operator is therefore unnecessary and can be safely removed.

For numeric primitive types, !! converts the source type to true if the value is non-zero. For reference types, !! converts the source type to true if the value is non-null. Neither of those is a no-op, so no warning is produced in those cases.

Example (flagged): 

disable="${!!active}"
After quick-fix: 
disable="${active}"

Reports a redundant !! (BOOLIFY) operator in an FXML binding expression when the resolved type of the bound path is Boolean.

The FXML compiler treats !! on a Boolean-typed observable as a compile-time no-op: the observable is passed directly to the target property without any BooleanBindings wrapping. The !! prefix is therefore unnecessary and can be safely removed.

For Number types !! maps to BooleanBindings.isNotZero, and for other reference types to BooleanBindings.isNotNull; neither of those is a no-op, so no warning is produced in those cases.

Example (flagged): 

disable="${!!active}"
After quick-fix: 
disable="${active}"

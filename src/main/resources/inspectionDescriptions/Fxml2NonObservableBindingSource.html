Reports FXML ${path} (Observe) and #{path} (Synchronize) binding expressions where no segment in the binding path resolves to a javafx.beans.value.ObservableValue.

When the entire path consists of plain fields or Java-Beans getters that do not return an ObservableValue, the FXML compiler rejects the binding with INVALID_UNIDIRECTIONAL_BINDING_SOURCE or INVALID_BIDIRECTIONAL_BINDING_SOURCE.

A quick-fix is offered to downgrade the binding to a one-time evaluate binding ($path).

Note: if any intermediate segment returns an ObservableValue (for example, the first segment is an ObjectProperty<ViewModel>), the binding is trackable and this inspection does not fire.

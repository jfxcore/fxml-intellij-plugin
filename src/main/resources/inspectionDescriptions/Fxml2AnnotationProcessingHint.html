Reports an unresolved {ClassName}Base reference in the extends clause of a @ComponentView-annotated class when the FXML compiler's annotation processor has not yet run.

The FXML compiler generates {ClassName}Base during annotation processing. Until a build with annotation processing enabled has been performed, this class does not exist and the extends reference cannot be resolved.

Example (base class not yet generated):

@ComponentView("""
    <StackPane/>
    """)
public class MainView extends MainViewBase {  // MainViewBase unresolved
    public MainView() {
        initializeComponent();
    }
}

Fix: rebuild the project with annotation processing enabled.

See the fxml-compiler documentation for setup instructions.

To suppress this warning, add @SuppressWarnings("Fxml2AnnotationProcessingHint") to the class.

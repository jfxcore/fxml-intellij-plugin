Reports constructors in FXML code-behind classes that do not call initializeComponent().

A class is a code-behind class when an FXML file exists in the project with fx:subclass pointing to that class. The compiler-generated base class provides initializeComponent(), which must be called in the constructor to instantiate the scene graph declared in the FXML file.

Example (problematic):

public class MyControl extends MyControlBase {
    public MyControl() {
        // initializeComponent() not called: scene graph never created
    }
}

Fix: call initializeComponent() in the constructor:

public class MyControl extends MyControlBase {
    public MyControl() {
        initializeComponent();
    }
}

To suppress this warning for a specific constructor or class, add @SuppressWarnings("Fxml2InitializeComponent").

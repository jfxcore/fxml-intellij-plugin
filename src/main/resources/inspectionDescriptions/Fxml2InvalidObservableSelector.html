Reports FXML binding path segments accessed via :: whose declared type is not a javafx.beans.value.ObservableValue subtype.

The :: operator is an observable-selection operator: it unwraps one level of ObservableValue so that the binding engine can track changes through the chain. When applied to a plain field or Java-Beans getter that does not return an ObservableValue, the FXML compiler rejects the expression with INVALID_INVARIANT_REFERENCE.

A quick-fix is offered to replace :: with ., converting the reference to a plain (non-trackable) property access. The fix is also available as a batch action ("Fix all in file" / "Fix all in project") via the Alt+Enter menu.

Note: the real-time error underline is provided by the FXML attribute annotator; this inspection surfaces the same problem so that the batch quick-fix is accessible.

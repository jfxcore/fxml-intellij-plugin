Reports element tags in FXML files that resolve to a generic class when no fx:typeArguments attribute is present.

When a generic class is instantiated without fx:typeArguments, the FXML compiler treats it as a raw type. Raw types bypass compile-time type safety and should be avoided.

Example (flagged):

<!-- ListView<T> has type parameters but no fx:typeArguments -->
<ListView/>

Corrected:

<ListView fx:typeArguments="String"/>

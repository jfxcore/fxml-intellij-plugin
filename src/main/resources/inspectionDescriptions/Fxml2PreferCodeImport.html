Reports <?import?> processing instructions inside embedded FXML markup (@ComponentView annotation values) that could be moved to the host Java or Kotlin file as regular import statements.

Keeping all imports in the host file's import section allows the IDE's import management (import optimizer, auto-import, unused-import detection) to handle every class reference uniformly, rather than having some imports hidden inside annotation string values.

The quick-fix Move to Java/Kotlin import adds the import to the host file and removes the <?import?> PI from the markup.

This inspection is disabled by default. Enable it when your project prefers all class references to be declared as regular code imports rather than as <?import?> PIs inside the markup.

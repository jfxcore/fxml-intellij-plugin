Creates a standalone FXML file from the markup embedded in a @ComponentView annotation on a Java or Kotlin class, and removes the annotation.

The action is available when the caret is on or inside the @ComponentView annotation (but not inside the annotation value string), provided that:

  • no sibling <ClassName>.fxml file already exists.

A confirmation dialog is shown before any changes are made. On confirmation:

  • A new FXML file named after the class (e.g. MyView.fxml) is created in the same directory as the Java or Kotlin source file.
  • The embedded markup is written to the FXML file with xmlns namespace declarations and fx:subclass restored on the root element.
  • Java or Kotlin import statements (excluding ComponentView itself) are converted to <?import?> processing instructions in the FXML file.
  • The @ComponentView annotation is removed from the class declaration.
  • The import org.jfxcore.markup.ComponentView statement is removed if no other @ComponentView annotation remains in the file.

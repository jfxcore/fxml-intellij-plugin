Embeds an FXML 2.0 file's markup directly into the code-behind class as a @ComponentView annotation and deletes the standalone FXML file.

The action is available when the caret is on the root element tag name, the fx:subclass attribute name, or the fx:subclass attribute value, provided that:

  • a code-behind sibling (.java or .kt) already exists next to the FXML file.

A confirmation dialog is shown before any changes are made. On confirmation:

  • The root element's markup (without xmlns:* and fx:subclass attributes) is inserted as a Java text-block or Kotlin multi-dollar-string @ComponentView("""...""") annotation before the class declaration.
  • The FXML processing instructions are added as Java or Kotlin import statements in the code-behind file, so the IDE's language injector can resolve all class references inside the embedded markup.
  • The FXML source file is deleted.

In Kotlin, a $$"""...""" multi-dollar string literal is used so that the $ character used in FXML 2.0 binding shortcuts (e.g. $vm.message) is treated as a literal rather than a template-interpolation prefix.
